The `use-ai-integration` skill's keyed (litellm/direct-HTTP) path now resolves credentials from the shared Claude `settings.json` env block at call time (`read_workspace_ai_credentials()` in `claude_p.py`) instead of `os.environ`, so services pick up workspace auth changes without restarts. The skill docs and `references/billing-and-credentialing.md` describe the settings-env credential contract.

Keyed integrations now pin their key at setup: the skill instructs snapshotting `ANTHROPIC_API_KEY` (+ `ANTHROPIC_BASE_URL`, never the subscription oauth token) into `runtime/secrets/anthropic.env` via the new `write_anthropic_env_snapshot()` helper, and `read_workspace_ai_credentials()` resolves that snapshot first (then the settings env, then the process env). A built service therefore keeps billing against the key it was set up with when the user later switches the workspace's sign-in; the user asks the agent to rewrite or delete the snapshot to re-key or retire integrations.
