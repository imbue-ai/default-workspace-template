- Mobile design for the workspace UI. On phone-sized screens (<=768px) the dockview tab strip -- tiny targets, wheel-driven overflow scrolling, unreachable overflowing tabs on touch -- is replaced by a mobile tab bar: a hamburger button at the top left opens a left-side drawer menu (with its own close button, plus backdrop tap to dismiss), with the active tab's title shown beside the button. The menu is one flat list of destinations (agent chats, terminal sessions, browsers, apps) rather than an open-tabs/open-new split: on a single-pane screen "open" is not a user-facing category, so tapping any row goes there (focusing the loaded panel or creating one). Loaded rows sort first (active on top) and carry a close X -- which doubles as the "currently loaded" signal -- plus a destroy action where the entity supports it; the creation and layout actions follow below. The desktop "+" dropdown keeps its existing not-yet-open filtering but adopts the same kind ordering (agents, terminals, browsers, apps). The app now sizes itself with dynamic viewport units (100dvh) instead of 100vh, so the composer no longer hides behind the mobile browser's retractable URL bar, and a visualViewport sync clamps the app above the on-screen keyboard on iOS Safari (where dvh does not shrink for the keyboard). Text inputs are 16px on small screens, below which iOS zooms the page on focus -- the zoom that used to shove the composer off screen. Transcript/composer padding compacts, dialogs size to the phone, and safe-area insets are respected (viewport-fit=cover).
- Fixed the dockview theme overrides silently not applying: dockview-core injects its stylesheet at runtime after the app bundle and re-applies the theme class on an inner element, so the app's `.dockview-theme-light` variable overrides (tab bar background, active-tab colors, drag-over accent) lost on cascade order and had been rendering dockview's defaults. The overrides are now anchored on the app-owned container class and actually take effect.
- Refactored the frontend stylesheet: the 3100-line style.css is split into styles/{theme,chat,composer,dockview,modals,responsive}.css with style.css as the import hub, and the empty-state overlay/dropdown JS inline styles moved into CSS classes.
