- Per-agent liveness dot on chat tabs: each agent's dockview tab shows a colored dot reflecting its mngr lifecycle state, distinct from the chat's activity indicator (which tracks work within a running process). The dot is green while the agent's claude process is up and working (RUNNING), yellow while the process is up but idle and waiting on the user (WAITING), and grey while the agent is dormant -- its process is not running (DONE, STOPPED, REPLACED, UNKNOWN) but revives on the next message. Hovering shows the exact lifecycle state. The state is carried on the agent WebSocket payload's `state` field (surfaced from mngr's discovery stream, which probes whether the expected claude process is alive in the agent's tmux pane), so the dot turns grey within one observe cycle when a process dies -- including when the OOM handler (earlyoom) sheds it. Every discovery event now carries a real probed state (mngr probes on incremental events too, not just full snapshots), so the interface records the reported state directly instead of assuming a just-surfaced agent is running -- a just-stopped agent no longer briefly reads as running. Where several states share a color (RUNNING/RUNNING_UNKNOWN_AGENT_TYPE are both green; DONE/STOPPED/REPLACED/UNKNOWN are all grey), the dot also takes a subtly different circular shape -- a solid disc for the expected state of each color (RUNNING, WAITING, DONE), a hollow ring for a process whose agent type is unconfirmed (RUNNING_UNKNOWN_AGENT_TYPE) or whose tmux window is gone (STOPPED), a ring with a center dot for a superseded process (REPLACED), and a faded disc for an indeterminate one (UNKNOWN) -- so the states stay tellable apart at a glance without relying on the tooltip.
